## O problema com valores mágicos

Quando um designer diz "o azul primário é #0066cc" e um desenvolvedor precisa traduzir isso para o código, algo invariavelmente se perde. Esse problema se multiplica quando a empresa cresce: cinco squads, dez repositórios, e o mesmo "azul primário" existe em oito variações diferentes espalhadas pelo código.

É aqui que entram os **design tokens**.

## O que são design tokens?

Design tokens são as variáveis atômicas de um sistema de design. Eles representam decisões de design — cor, tipografia, espaçamento, elevação — de forma agnóstica à plataforma.

```json
{
  "color": {
    "primary": { "value": "#0066cc" },
    "primary-hover": { "value": "#0055aa" }
  },
  "spacing": {
    "sm": { "value": "0.75rem" },
    "md": { "value": "1.5rem" }
  }
}
```

## Por que tokens e não apenas variáveis CSS?

Variáveis CSS são uma implementação. Tokens são a fonte da verdade. A diferença:

- **Tokens** existem no Figma, em JSON, em Swift, em Kotlin — em qualquer plataforma
- **Variáveis CSS** existem apenas no browser
- Tokens permitem **sincronização automática** entre design e código via ferramentas como Style Dictionary ou Token Transformer

## Os três níveis de tokens

Uma taxonomia bem definida separa os tokens em três camadas:

### 1. Tokens primitivos (Global)

São os valores brutos. Nunca usados diretamente nos componentes.

```
color.blue.500 = #0066cc
color.blue.600 = #0055aa
```

### 2. Tokens semânticos (Alias)

Dão significado aos tokens primitivos. Respondem à pergunta "para que serve?".

```
color.action = color.blue.500
color.action-hover = color.blue.600
```

### 3. Tokens de componente

Específicos para um componente. Apontam para tokens semânticos.

```
button.primary.background = color.action
button.primary.background-hover = color.action-hover
```

## Resultado na prática

Quando migramos para tokens no design system da [Empresa A], o time de design conseguiu refatorar o tema de 8 horas de trabalho para 15 minutos — só atualizando os tokens primitivos. Tudo o mais cascateou automaticamente.

Essa é a beleza dos tokens bem estruturados: **mude em um lugar, propague em todos**.
